# SciTeX Medium Strategy

## Content Types

### 1. Technical Deep-Dives
Long-form explanations of SciTeX components and design decisions.

```
Title: [Component Name]: How [Problem] Is Solved in SciTeX

Structure:
- The Problem (2-3 paragraphs)
- Existing Solutions & Limitations
- Our Approach
- Technical Implementation
- Results/Impact
- What's Next
```

### 2. Build-in-Public Updates
Monthly progress summaries with lessons learned.

```
Title: Building SciTeX: [Month] Update

Structure:
- Overview (what shipped)
- Challenges faced
- Key learnings
- Community feedback
- Next month's focus
```

### 3. Research Workflow Tutorials
Practical guides for researchers.

```
Title: How to [Specific Task] with SciTeX

Structure:
- Prerequisites
- Step-by-step guide
- Code examples
- Tips & gotchas
- Further reading
```

## Guidelines
- 5-10 minute read optimal
- Include code snippets with syntax highlighting
- Add visuals (diagrams, screenshots, GIFs)
- Cross-link to GitHub/documentation
- End with clear CTA (try it, star repo, join discussion)

## Tags
`open-source` `research` `scientific-computing` `python` `latex` `data-visualization` `academia` `reproducibility`

## Publishing Schedule
- 1-2 posts per month (quality over quantity)
- Cross-post to Note.com for Japanese audience

## Tracking
Log each post in `posted/YYYY-MMDD-medium/`
