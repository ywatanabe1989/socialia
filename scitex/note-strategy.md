# SciTeX Note Strategy (note.com)

## Purpose
Reach Japanese researchers and developers with localized content.

## Content Types

### 1. 技術解説 (Technical Explanations)
SciTeXのコンポーネントと設計思想の詳細解説。

```
タイトル: [コンポーネント名]：SciTeXにおける[課題]の解決方法

構成:
- 課題の背景
- 既存ソリューションの限界
- SciTeXのアプローチ
- 技術的実装
- 結果・効果
- 今後の展望
```

### 2. 開発日記 (Development Diary)
Monthly progress in Japanese.

```
タイトル: SciTeX開発記録：[月]の進捗

構成:
- 今月のリリース内容
- 直面した課題
- 学んだこと
- コミュニティからのフィードバック
- 来月の予定
```

### 3. 研究ワークフロー入門 (Workflow Tutorials)
日本の研究者向け実践ガイド。

```
タイトル: SciTeXで[具体的なタスク]を行う方法

構成:
- 前提条件
- ステップバイステップガイド
- コード例
- Tips & 注意点
- 参考リンク
```

## Guidelines
- Write in Japanese (日本語で執筆)
- 5-10 minute read
- Include code with comments in English (code is universal)
- Explain Western OSS concepts for Japanese context
- Link to Japanese documentation when available

## Tags
`オープンソース` `研究` `科学計算` `Python` `LaTeX` `データ可視化` `学術` `再現性`

## Publishing Schedule
- Translate/adapt key Medium posts
- 1 post per month minimum
- Publish after Medium version (allows iteration)

## Cross-posting Strategy
Medium → Note (adapt, don't just translate)
- Adjust examples for Japanese research context
- Add Japanese-specific tools/integrations
- Reference Japanese academic conventions

## Tracking
Log each post in `posted/YYYY-MMDD-note/`
