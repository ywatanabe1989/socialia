# Note 投稿フォーマット (日本語ブログ)

## 構造

```
[WHY]   気づき - 共感を呼ぶ視点、「確かに」と思わせる
        ↓
[PAIN]  不便 - 研究者あるある、日常の面倒
        ↓
[HOW]   解決 - 「だから作った」という自然な流れ
        ↓
[CODE]  デモ - 日本語コメント付き
        ↓
[FEATURES]  機能まとめ
        ↓
[CTA]   フィードバック歓迎
```

## テンプレート

```
# [モジュール名] - [一行説明]

## なぜ作ったか

[普遍的な気づき]
例: MacやWindowsでは、ファイルをダブルクリックすれば適切なアプリが開く。

[不便さ]
例: 研究では毎日さまざまなファイル形式を扱う。CSV、JSON、NPY、PTH... それぞれ別のAPIを覚えないといけない。

[解決への動機]
例: コードでも同じ「クリックすれば開く」感覚が欲しかった。
```

## 制約

- Note は長文OK、丁寧な説明が好まれる
- 「なぜ作ったか」セクションで動機を明確に
- 技術記事というより「個人の試行錯誤」トーン
- 最後に「より大きなプロジェクト」を匂わせる（SciTeX発見への導線）
