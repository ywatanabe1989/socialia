# Medium Post Format (English)

## Structure

```
TITLE:  [Catchy but honest - no clickbait]
SUBTITLE: [One-line value prop]

[WHY]   Opening hook - universal insight
        ↓
[PAIN]  The problem - relatable frustration
        ↓
[HOW]   The solution - "So I built..."
        ↓
[CODE]  Multiple examples, well-commented
        ↓
[FEATURES]  Detailed breakdown
        ↓
[CLOSING]  Personal reflection + CTA
```

## Template

```
# [Catchy Title About the Problem]
## [One-line description of the solution]

[Universal insight - 1-2 sentences that make reader nod]

[Pain point - expand on why this is frustrating. Personal angle works well on Medium.]

[Transition to solution - "So I decided to..." or "After writing the same code for the Nth time..."]

## The Solution

**scitex[module]** - [one-line description]

\`\`\`python
from scitex.xxx import yyy

# Example 1: Basic usage
# ...

# Example 2: Advanced usage
# ...
\`\`\`

## What It Supports

- **Category**: item, item, item
- **Category**: item, item, item

## Key Features

1. **Feature 1** - brief explanation
2. **Feature 2** - brief explanation
3. **Feature 3** - brief explanation

## Installation

\`\`\`bash
pip install scitex[module]
\`\`\`

## Closing Thoughts

[Personal reflection - why this matters to you]

[Soft CTA - "If you've felt the same frustration, give it a try. Feedback welcome."]

---

*This is part of a larger research toolkit I'm building. More modules coming.*
```

## Constraints

- Medium readers expect polish and narrative
- Code examples should be comprehensive but not overwhelming
- Personal story angle performs well
- Avoid "10x" or "Revolutionary" language
- End with forward-looking statement (seeds SciTeX)
- Tags: Python, Programming, Data Science, Research, Open Source
