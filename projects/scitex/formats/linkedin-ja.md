# LinkedIn 投稿フォーマット (日本語)

## 構造

```
[WHY]   1-2行 - 誰もが共感する気づき
        ↓
[PAIN]  2-3行 - 日常の不便、個人的な視点
        ↓
[HOW]   1段落 - 「だから作った」という自然な流れ
        ↓
[CODE]  8-12行、コメント付き
        ↓
[FEATURES]  「など」付きのインラインリストまたは短い箇条書き
        ↓
[CTA]   控えめに - 「フィードバック歓迎」
        ↓
[HASHTAGS]  3-4個の関連タグ
```

## テンプレート

```
[普遍的な気づき - 他ではこう動くのに]

[日常の不便さ - 「毎回〜するのが面倒だった」という個人的な視点]

だから **scitex[module]** を作った。[一行説明]

\`\`\`python
from scitex.xxx import yyy

# コメント付きの例
\`\`\`

[機能まとめ: X / Y / Z など]

pip install scitex[module]

自分のワークフロー用に作ったもの。同じ不便を感じていたらフィードバック歓迎。

#Python #Research #DataScience #OpenSource
```

## 制約

- 最初の2-3行で読むか決まる
- 「など」で列挙疲れを防ぐ
- 個人視点 > 一般論
- 売り込み感を出さない
